# UI/UX 및 프론트엔드 개발 가이드라인

이 문서는 프로젝트의 UI 일관성을 유지하기 위한 공식 가이드라인입니다. 모든 프론트엔드 개발 및 수정을 수행할 때 이 규칙을 준수해야 합니다.

## 1. 리스트 및 테이블 규칙 (List & Table)

### 1-1. 항목 넘버링 (Row Numbering)
- 모든 리스트 테이블의 첫 번째 열은 **고유 순번(No.)**을 표시해야 합니다.
- 순번은 데이터의 ID가 아닌, 현재 페이지와 정렬 상태를 기준으로 계산된 사용자 가독 전용 번호를 사용합니다.
- **계산 공식**: `(현재페이지 - 1) * 페이지당항목수 + (인덱스 + 1)`
- 역순 표시가 필요한 경우: `전체항목수 - ((현재페이지 - 1) * 페이지당항목수 + 인덱스)`

### 1-2. 페이징 및 필터 (Pagination & Filter)
- 리스트 하단에는 반드시 페이징 컴포넌트가 포함되어야 합니다.
- **페이징 구성**: `[이전]` `1`, `2`, `3` ... `[다음]` 형태를 유지합니다.
- **행 수 선택 (Rows per Page)**: 우측 또는 좌측 하단에 페이지당 보여줄 행 수를 선택하는 `Select` 박스를 배치합니다. (기본 옵션: 10, 20, 50, 100개씩 보기)

## 2. 버튼 스타일 가이드 (Buttons)

모든 버튼은 Shadcn UI의 `Button` 컴포넌트와 `variant`를 사용하여 의미론적으로 통일합니다.

| 역할 | Variant | 색상/특징 | 예시 |
| :--- | :--- | :--- | :--- |
| **주요 액션** | `default` | 검정색/진한 회색 (Primary) | 저장, 등록, 확인 |
| **보조 액션** | `outline` | 테두리만 있음 | 취소, 돌아가기, 목록 |
| **위험 액션** | `destructive` | 빨간색 | 삭제, 경고 |
| **아이콘/링크** | `ghost` | 배경 없음, 호버 시 노출 | 삭제 아이콘, 수정 아이콘 |
| **기타/선택** | `secondary` | 연한 회색 | 임시저장, 필터 초기화 |

## 3. 아이콘 및 타이포그래피 (Icons & Typography)
- 모든 아이콘은 `lucide-react` 라이브러리를 사용합니다.
- 텍스트와 아이콘이 함께 있는 경우, 아이콘과 텍스트 사이에 `gap-2` (8px)의 간격을 유지합니다.
- 모든 금액 표시에는 원화 기호(`₩`)와 천 단위 콤마(`.toLocaleString()`)를 적용합니다.
