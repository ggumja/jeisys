# Jeisys Medical B2B 쇼핑몰 구축 플랫폼 개요

본 문서는 현재까지 구축된 **Jeisys Medical B2B 의료기기 및 소모품 쇼핑몰 플랫폼**의 전체 기능과 시스템 아키텍처를 소개합니다. 병원(고객)의 구매 편의성을 극대화하고, 영업사원(대리인) 및 본사 관리자의 효율적인 운영을 지원하기 위한 End-to-End E-commerce 시스템입니다.

---

## 1. 병원 / 의사 (사용자) 주요 기능

### 1-1. 상품 및 쇼핑 특화 기능
* **의료장비 기반 상품 검색:** 병원이 보유한 의료기기(Equipment)와 호환되는 전용 소모품 및 부자재를 자동으로 필터링하여 제공
* **다양한 상품 구성:** 단일 상품(Product)뿐만 아니라 복합 패키지(Package) 및 세트(Set) 상품 구매 지원
* **빠른 주문 (Quick Order):** 병원의 과거 구매 이력을 바탕으로 소모품을 한 번에 재주문할 수 있는 원클릭 시스템
* **정기 배송 (Subscription):** 주기적으로 필요한 의료 소모품을 특정 간격(예: 30일, 60일)으로 자동 결제 및 배송하는 기능

### 1-2. 결제 및 크레딧 시스템 (고도화)
* **장비별 전용 크레딧 (Equipment Credit):** 장비 구매 시 제공받은 크레딧을 해당 장비의 소모품 구매에만 사용할 수 있도록 제한/관리하는 스마트 크레딧 로직
* **복합 결제 (Split Payment) 지원:** B2B 거래의 특성(카드 한도 초과 등)을 고려하여 총 주문 금액을 최대 5개의 결제 수단으로 나누어 결제
    * *제공 수단:* 등록된 법인카드 (간편결제), 일반 신용카드 (PG 결제창), 가상계좌 (무통장 입금)
* **자동 잔액 계산:** 복합 결제 시 남은 금액을 실시간으로 계산하고 원클릭으로 잔액을 특정 카드에 할당하는 UI 제공

### 1-3. 주문 및 마이페이지
* **Self-Service 주문 취소:** '결제 완료' 및 '배송 준비 중' 상태의 주문에 대해 고객이 직접 취소 사유를 입력하고 환불(크레딧 복구 등)을 처리할 수 있는 자동화 시스템
* **다중 배송지 분할 (Split Shipping):** 대량 주문 시 주문한 품목을 여러 배송지(지점)로 나누어 배송 요청 가능
* **거래명세서 및 영수증:** 결제 내역에 대한 세금계산서 및 거래명세서 즉시 발급 및 조회

### 1-4. 고객 지원 및 커뮤니케이션
* **장비 데모 및 교육:** 신규 의료기기 데모(Demo) 시연 신청 및 오프라인/온라인 교육(Education) 세미나 예약 시스템
* **자료실:** 장비별 고해상도 이미지, 작동 매뉴얼(PDF), 임상 영상(Media) 스트리밍 제공
* **1:1 문의 및 공지:** 뉴스/언론보도, 공지사항, FAQ 및 실시간 1:1 고객 문의 게시판

## 2. Admin (관리자 및 영업사원) 주요 기능

### 2-1. 주문 및 영업 관리
* **대리 주문 (Proxy Cart):** 영업사원이 관리자 페이지에서 특정 병원의 장바구니를 대신 구성하고, 견적을 발송하거나 직접 주문/결제를 대행하는 기능
* **주문 상태 및 배송 관리:** 전체 주문의 라이프사이클(접수, 입금확인, 배송처리, 교환/반품/취소) 중앙 제어
* **매출 통계 대시보드:** 기간별 매출 통계, 지역/영업소별 실적, 상품별 판매 추이를 한눈에 파악하는 시각화 대시보드

### 2-2. 상품 및 카탈로그 제어
* **상세한 상품 메타데이터 관리:** 단가, 재고, 옵션, 장비 호환성, 프로모션 뱃지 등을 제어하는 PIM(Product Information Management) 기능
* **광고 배너 및 메인 페이지 제어:** 사용자 메인 홈페이지의 롤링 배너, 추천 상품 큐레이션, 이벤트 영역 등을 노코드 형태로 관리

### 2-3. 회원 및 영업망 관리
* **병원(Member) 상세 제어:** 가입 승인, 상태 변경, 장비 보유 현황 수동 매핑
* **크레딧 수동 제어:** 관리자가 병원 측에 특정 사유로 크레딧을 지급하거나, 오지급된 크레딧을 회수(차감) 및 이력 추적
* **영업 조직 관리:** 영업소(Sales Office) 생성 및 영업사원(담당자)별 병원 할당 기능

## 3. 핵심 아키텍처 및 기술 스택 (Tech Stack)

### 3-1. Frontend (클라이언트)
* **프레임워크:** React 18, TypeScript, Vite
* **UI 컴포넌트 및 스타일링:** Tailwind CSS, Radix UI 기반의 Headless 접근성 컴포넌트 활용
* **디자인 시스템:** Vercel UI 가이드라인을 참조한 모던 대시보드 및 글래스모피즘(Glassmorphism) 트렌드 적용
* **상태 관리 및 데이터 페칭:** Zustand (Client State), React Query (Server State / Caching)

### 3-2. Backend 및 인프라 (Supabase)
* **데이터베이스:** PostgreSQL 기반의 Supabase DB 사용
* **보안 및 인증:** Row Level Security (RLS) 정책을 통해 병원(고객)과 관리자(Admin) 간의 엄격한 데이터 접근 통제 구현, Supabase Auth를 통한 세션 관리
* **스토리지:** 제품 이미지, 썸네일, 매뉴얼 PDF, 동영상 등을 서빙하는 Supabase Storage CDN 연동
* **Rich Text 연동:** 공지사항, 상품 상세설명, 뉴스 등을 작성하기 위한 TipTap 기반의 WYSIWYG 에디터 구현

### 3-3. 비즈니스 로직 고도화 포인트
* **Atomic Transactions:** 주문 취소 시 재고 롤백, 결제 상태 변경, 장비 크레딧 환불, 취소 사유 로깅 등이 하나의 트랜잭션으로 원자적으로(Atomic) 처리되도록 SQL 프로시저 및 RLS 적용
* **유연한 결제 연동 기반:** KICC EasyPay (간편결제/일반결제/가상계좌) PG 모듈과 연동 가능한 주문 Payload 아키텍처 (배열 형태의 다중 결제 요청 처리)
