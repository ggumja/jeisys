# 상품 유형별 테스트 시나리오 가이드

본 문서는 데이터베이스 초기화 후 상품 등록부터 주문 완료까지의 전체 프로세스를 검증하기 위한 테스트 시나리오입니다.

---

## 1. 일반 단일 상품 (Standard Product)
기본적인 판매 정보와 수량별 할인이 적용되는 상품을 테스트합니다.

### 1-1. 상품 등록 (Admin)
- [ ] 상품명, 카테고리, 상품코드(SKU), 기본 가격 입력이 정상적인가?
- [ ] 대표 이미지 및 추가 이미지 업로드가 성공하는가?
- [ ] **대량 구매 할인(Pricing Tier)**을 설정하고 저장되는가? (예: 10개 이상 10% 할인)
- [ ] 상세 설명(에디터) 내용이 정상적으로 저장되는가?

### 1-2. 고객 경험 (Customer)
- [ ] 상품 상세 페이지에서 기본 가격과 이미지, 설명이 잘 보이는가?
- [ ] 수량을 대량으로 변경했을 때 할인된 금액이 실시간으로 하단 총 금액에 반영되는가?
- [ ] 장바구니 담기 및 주문서에서 할인된 단가가 유지되는가?

---

## 2. 세트 옵션 및 증정 상품 (Set Options & Bonus Items)
수량별 전용 옵션과 그에 따른 추가 증정품이 포함된 상품을 테스트합니다.

### 2-1. 상품 등록 (Admin)
- [ ] **수량 옵션 사용** 활성화 후 세트별 가격을 다르게 설정할 수 있는가? (예: 2개입 세트 - 15% 할인)
- [ ] 각 수량 옵션별로 **추가 증정 상품**을 검색하여 매핑할 수 있는가?
- [ ] 저장 후 다시 수정 페이지에 진입했을 때 설정한 옵션과 증정품 목록이 유지되는가?

### 2-2. 고객 경험 (Customer)
- [ ] 상품 상세에서 세트 옵션(드롭다운 혹은 라디오)이 정상 노출되는가?
- [ ] 특정 옵션 선택 시 하단에 **증정 상품 정보**가 팝업 또는 텍스트로 안내되는가?
- [ ] 장바구니에 담았을 때 본품과 증정품이 함께 세트로 묶여서 보이는가?

---

## 3. 패키지 상품 (Package Product)
여러 상품을 하나로 묶어 판매하며, 고객이 구성품을 선택할 수 있는 상품을 테스트합니다.

### 3-1. 패키지 등록 (Admin)
- [ ] 구성 상품을 검색하여 추가하고, 각 상품별 **가격 오버라이드**가 가능한가?
- [ ] **선택 가능 상품 수**를 설정하여 고객의 선택 폭을 제한할 수 있는가?
- [ ] 패키지 자체의 재고 관리가 정상적으로 작동하는가?

### 3-2. 고객 경험 (Customer)
- [ ] 패키지 전용 상세 페이지에서 구성 상품들을 선택(체크박스 등)할 수 있는가?
- [ ] 지정된 선택 개수를 초과하거나 미달할 경우 안내 메시지가 표시되는가?
- [ ] 주문 완료 후 마이페이지 주문 내역에서 패키지 내부 구성품 목록이 정확히 표시되는가?

---

## 4. 공통 E2E 테스트 (End-to-End)
프로세스 전체의 데이터 정합성을 확인합니다.

- [ ] **주문 관리 (Admin)**: 고객이 주문한 내역이 관리자 페이지 `Order Management`에 즉시 나타나는가?
- [ ] **데이터 정합성**: 주문 상세 보기에서 옵션 할인액, 증정품 정보, 최종 결제 금액이 고객이 본 화면과 일치하는가?
- [ ] **재고 차감**: 주문 완료 후 상품 및 구성품의 재고가 설정값에 따라 차감되는가?
