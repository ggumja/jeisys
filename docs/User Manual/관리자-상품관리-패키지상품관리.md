# 📦 복합 패키지 상품 등록 및 운영 매뉴얼 (v2.0)

이 문서는 제이시스 소모품 쇼핑몰의 **복합 구성 패키지 상품**을 관리자가 정확하고 효율적으로 등록/수정하기 위한 통합 가이드입니다. 개편된 UI와 자동 검증 로직을 포함하고 있습니다.

---

## 🚀 1. 메뉴 접속 및 기본 정보
**메뉴 경로**: `관리자 > 상품 관리 > 패키지 상품 등록(또는 수정)`

### 🛒 1.1 기본 정보 설정
*   **대표 이미지**: 패키지 상품의 메인 이미지를 업로드합니다 (권장: 800x800px).
*   **패키지 명**: 고객에게 노출될 상품명을 입력합니다.
*   **상품 카테고리**: 해당 패키지가 속할 카테고리를 선택합니다.
*   **상품 코드 (SKU) / SAP SKU**: 내부 관리용 코드를 입력합니다.
*   **재고 수량**: 패키지 전체의 판매 가능 재고를 입력합니다.
*   **판매 상태**: '판매중' 또는 '일시품절'을 설정합니다. (재고가 우선 노출되도록 UI가 최적화되어 있습니다.)

> **[참고] 가격(판매가) 자동 동기화**: 패키지 전체의 대표 가격은 하단의 '패키지 상품 구성 옵션' 중 가장 낮은 금액으로 자동 설정됩니다. 별도로 입력하실 필요가 없습니다.

## 🍱 2. 패키지 상품 구성 옵션 (핵심 설정)
이 섹션은 패키지의 종류(예: 20개 세트, 50개 세트)와 그 안에 담길 구성 상품들을 정의하는 가장 중요한 구역입니다.

### 3.1 옵션 카드 설정
1.  **[옵션 추가]** 버튼을 클릭하여 새 옵션 카드를 만듭니다.
2.  **옵션 표기명**: 고객이 선택할 세트 이름 (예: `20개 혼합 패키지`)
3.  **패키지 총 구성 갯수**: 해당 세트에 포함되어야 할 **실제 제품의 총 수량** (예: `20`)
4.  **패키지 확정 상품금액**: 해당 세트 전체의 판매 가격 (예: `4,000,000`)

### 3.2 실시간 수량 검증 가이드 (Selection Progress)
검색창 바로 아래에 나타나는 **실시간 안내 바**를 확인하십시오.
*   **메세지**: `총 N개의 상품을 선택하세요. 현재 M/N 개가 선택되었습니다.`
*   **작동 원리**: 하단 구성 상품 테이블의 **'최대 선택 수량'** 칸에 입력된 숫자들의 합계를 실시간으로 계산합니다.
*   **성공 피드백**: 현재 수량(M)이 목표 수량(N)과 정확히 일치하면 숫자가 변화하며 활성화된 상태임을 알려줍니다.

### 3.3 구성 상품 검색 및 추가
*   **상품 검색**: '구성 상품명 또는 SKU로 검색' 칸에 입력 후 검색 결과에서 상품을 클릭하면 즉시 추가됩니다.
*   **선택 후 자동 비움**: 상품을 선택하면 검색창의 텍스트가 자동으로 사라져 연속 검색이 편리합니다.
*   **최대 선택 수량 설정**: 각 구성 상품 우측의 `최대 선택 수량` 칸에 숫자를 입력하십시오. 이 수량들의 총합이 위에서 설정한 '패키지 총 구성 갯수'와 일치해야 합니다.

## 🎁 3. 세트 추가 제공 상품 (보너스)
특정 옵션을 선택했을 때 덤으로 증정할 상품을 설정합니다.

1.  **상품 검색**: 증정할 상품을 검색하여 추가합니다.
2.  **계산 방법**:
    *   **직접 입력 (Fixed)**: 구매 수량과 상관없이 정해진 개수만 증정합니다.
    *   **비율 계산 (Ratio)**: 고객의 구매 수량에 비례하여 증정품 개수를 자동 계산합니다 (예: 100% 설정 시 구매 1개당 증정 1개).

## 📝 4. 상세 설명 및 등록 완료
*   **상품 설명**: Rich Text Editor를 사용하여 이미지와 텍스트로 상세 정보를 구성합니다.
*   **최종 전송**: 하단의 `[등록 완료]` 또는 `[수정 완료]`를 클릭합니다.

> **[팁] 간헐적 로딩 이슈 해결됨**: 이전 버전에서 수정 페이지 진입 시 구성 상품 목록이 가끔 보이지 않던 문제는 데이터 초기화 가드 로직이 적용되어 현재 완벽히 해결되었습니다 (v2.0 반영). 안정적으로 편집을 진행하실 수 있습니다.

## 💡 자주 묻는 질문 (FAQ)
**Q: 사용자 화면에서 금액이 0원으로 보여요.**
A: 패키지 상품은 고객이 **'구매 세트 선택'**에서 옵션을 선택하기 전까지는 `₩0`으로 표시되도록 설계되었습니다. 선택 직후 확정 금액이 나타납니다.

**Q: 구성 상품 테이블에 상품을 추가했는데 상단 카운트가 안 올라가요.**
A: 품목만 추가해서는 카운트되지 않습니다. 해당 행(Row)의 **'최대 선택 수량'** 칸에 숫자를 입력해야 비로소 합계에 반영됩니다.

**Q: '정상가(개당)' 정보는 왜 안 보이나요?**
A: 패키지 상품은 단품 판매가 아닌 세트 단위 결제이므로, 고객의 혼선을 방지하기 위해 개당 단가 노출을 의도적으로 숨겼습니다.
