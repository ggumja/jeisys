# PDCA - Admin Dashboard Real Data Integration (Act)

## Act (사후 조치 및 개선점)
- 현재 이번 달에 판매된 `order_items`의 총 개수가 많은 경우 100건 단위 청크(Chunk)로 나누어 요청하도록 개발되어, API 호출 지연을 완화했습니다. 만약 일일 매출액이나 주문 수가 기하급수적으로 폭증할 경우, 이 데이터를 캐싱하거나 통계 테이블을 따로 분리해 야간 스케줄러(Cron)가 주기적으로 집계하도록 DB 아키텍처를 추후 고도화할 필요가 있습니다.
- 관리자 권한을 가진 유저가 대시보드 로딩 시 모든 통계를 실시간 연산하므로, 체감 속도를 높이기 위한 캐싱 전략 도입을 고려합니다.
