# 복합결제 UI 구현 내역 (Act)

고객이 한 번의 주문에서 여러 장의 카드 또는 카드와 가상계좌를 섞어서 결제할 수 있도록 지원하는 "복합결제" UI 기능을 추가했습니다. 

## 1. 구현 내용

### 프론트엔드 UI/UX 변경 (`src/pages/CheckoutPage.tsx`)
- **결제 탭 분리:** `단일 결제`와 `복합 결제` 탭을 도입하여, 사용자 선택에 따라 인터페이스가 전환되도록 구현.
- **정기배송 제약 조건 처리:** 장바구니에 정기배송 상품이 있는 경우, 복합결제 탭을 진입 불가 상태로 잠그고 안내 메시지를 표시 (빌링키 단일 결제 필요).
- **동적 입력 행 (Dynamic Rows):** 결제 방식을 최대 5개까지 추가할 수 있는 "+ 결제 수단 추가" 기능 적용. 
  - 각 행에서 법인카드/가상계좌 드롭다운 선택.
  - 신용카드 선택 시 등록된 카드의 선택을 지원.
  - 행 삭제 기능 제공.
- **금액 배분 검증 로직:** "잔액 적용" 버튼으로 남은 금액을 쉽게 털어낼 수 있도록 지원. 최종 남은 결제 금액이 정확히 `0원`일 때만 주문 완료 버튼이 활성화되도록 방어 로직 구현.

### 백엔드 데이터 전송 규격 (타입/서비스)
- `src/types/index.ts`: `SplitPaymentMethod` (id, type, cardId, amount) 인터페이스 추가.
- `src/services/orderService.ts`: `createOrder` 함수의 `OrderInput` 구조에 `splitPayments` 배열 파라미터 추가.

## 2. 향후 백엔드 연동 계획 (Follow-up Task)
현재 백엔드의 `orderService.createOrder` 내부에서는 단일 `paymentMethod` 기준으로 승인 처리가 이루어집니다. 프론트엔드가 생성한 `splitPayments` 배열 정보를 기반으로 PG(KICC) 다중 승인을 지원하도록, 추후 백엔드 API 보강 작업이 필요합니다.
