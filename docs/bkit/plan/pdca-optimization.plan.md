# 계획: PDCA 가동 규칙 최적화 (토큰 절약형)

## 목적
무분별한 문서 생성을 지양하고, 토큰 효율성을 높이기 위해 트리거 기반의 PDCA 워크플로우를 도입함.

## 범위
- `docs/bkit` 아카이빙 정책 변경.
- `AGENTS.md` 및 `MEMORY.md` 규칙 갱신.

## 요구사항
- **/pdca** 접두어가 포함된 명령에 대해서만 `docs/bkit` 문서 생성.
- 중요도가 높은 작업이나 설계 변경의 경우 사용자 확인 후 수동 가동.

## 성공 기준
- 작업 속도 향상 및 토큰 소모량 감소.
- 필수적인 이력만 `docs/bkit`에 정제되어 저장됨.
