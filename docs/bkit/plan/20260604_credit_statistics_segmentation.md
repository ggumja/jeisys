# 크레딧 통계 분석 세분화 및 실시간 데이터 연동 계획

어드민 통계분석 메뉴에 '크레딧 통계(`/admin/statistics/credits`)' 서비스를 새롭게 추가하고, Supabase `user_credits` 및 `credit_transactions` 테이블 데이터를 연동하여 장비별/회원별 실시간 크레딧 현황을 시각화하기 위한 계획입니다.

## 1. 크레딧 통계 화면 제안

매출 및 상품 분석과 일관된 UI(상단 인라인 세그먼트 기간 필터, 탭 구조, 브랜드 컬러 `#21358D` 적용)를 적용하여 아래와 같은 4가지 세부 분석 화면을 제안합니다.

### 📌 탭 1. 크레딧 개요 (`overview`)
* **핵심 지표 카드**: 누적 총 발행액, 누적 총 사용액, 현재 총 잔액(Remaining), 누적 총 만료액.
* **장비별 잔액 분포 (Pie Chart)**: 각 장비(Density, POTENZA 등)가 전체 크레딧 잔액에서 차지하는 비중 시각화.
* **월별 발행 vs 사용 추이 (라인/막대 혼합 차트)**: 최근 6개월간 월별 신규 크레딧 발행액과 실제 사용액의 비교 변동 추이 렌더링.

### 📌 탭 2. 장비별 통계 (`equipment`)
* **장비별 성과 테이블**: 장비 모델명(Density, POTENZA 등) 기준 발행 총액, 사용 총액, 소진율(%), 남은 잔액, 활성 상태의 크레딧 건수 표시.
* **장비별 주요 보유 고객**: 특정 장비의 크레딧 잔액을 많이 보유하고 있는 상위 병원(고객) 랭킹 리스트 제공.

### 📌 탭 3. 만료 임박 분석 (`expiry`)
* **만료 임박 요약**: 향후 30일 / 60일 / 90일 이내에 유효기간이 만료되는 크레딧의 총 잔액 및 대상 병원(고객) 수 집계.
* **만료 임박 상세 리스트 (Table)**: 30일 이내 만료 예정인 크레딧 정보를 테이블 형식으로 표시 (병원명, 대표자, 장비명, 남은 금액, 만료 예정일, 고객 연락처 등). 마케팅/프로모션 대상 선정을 돕습니다.

### 📌 탭 4. 트랜잭션 통계 (`transactions`)
* **거래 유형별 집계**: 발행(issue), 사용(use), 환불(refund), 회수(revoke), 만료(expire) 등 각 트랜잭션 타입별 월간 거래 건수 및 누적 금액 변동 현황 분석.
* **크레딧 충전/소진 주기 분석**: 발행 후 최초 사용 시점까지의 평균 소요 시간 및 완전 소진까지의 평균 리드 타임 통계 제공.

## 2. 데이터 연동 계획 (Backend API)

`adminService.ts` 에 크레딧 통계 조회를 위한 다음 API들을 추가 정의합니다.

1. `getCreditOverviewStats(dateRange: string)`
   * 지정 기간 내 `credit_transactions` 데이터를 집계하여 누적 발행/사용/만료액 산출.
   * `user_credits` 장비별 잔액 합계 연산.
2. `getCreditEquipmentStats(dateRange: string)`
   * 장비별 발행/사용/잔액을 집계하고, 해당 장비의 상위 보유 고객 랭킹을 반환.
3. `getCreditExpiryStats(limitDays: number)`
   * `expiry_date` 필드를 기준으로 만료 임박액 요약 데이터 및 상세 리스트 반환.
4. `getCreditTransactionStats(dateRange: string)`
   * 트랜잭션 타입별 통계 추이 데이터 반환.

## 3. 검증 계획

### 자동 및 수동 검증
* Supabase `user_credits` 및 `credit_transactions` 테이블의 조인 및 RPC 함수가 정상적으로 정합성 높은 통계 데이터를 산출하는지 DB 쿼리 로그 검증.
* 상단 인라인 세그먼트 기간 필터(최근 7일, 최근 30일, 기간 지정 등) 변경 시 차트와 테이블이 유연하게 갱신되는지 확인.
* UI 가이드라인에 따른 리스트 넘버링 공식, 페이징 컴포넌트, 브랜드 컬러(`#21358D`) 정합성 확인.
