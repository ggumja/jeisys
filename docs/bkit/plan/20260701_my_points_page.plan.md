# PDCA Plan: 마이페이지 내 마이 포인트 메뉴 추가 및 내역 조회 기능 구현

## 1. 개요 (Plan)
사용자 마이페이지에 '마이 포인트' 메뉴를 신설하여 사용자가 본인의 현재 포인트 잔액, 총 적립액, 총 사용액을 확인하고 적립/사용/환불/회수/만료 등의 상세 변동 이력을 시간순으로 조회할 수 있는 페이지를 구현합니다.

## 2. 세부 사양 (Design)

### A. 메뉴 및 라우팅 추가
- **메뉴 경로**: `/mypage/points`
- **마이페이지 레이아웃 추가**: `MyPageLayout.tsx` 내 네비게이션 메뉴에 '마이 포인트' 항목 추가 (아이콘: `Award` 또는 `Coins`)
- **라우터 등록**: `routes.tsx` 내 `mypage` 자식 경로 아래에 `points` 패스로 `MyPointsPage` 컴포넌트 매핑

### B. 마이 포인트 페이지 UI/UX 구성 (`MyPointsPage.tsx`)
- **포인트 요약 카드**:
  - **현재 보유 포인트**: 가장 강조하여 표시 (예: `50,000 P`)
  - **누적 적립 포인트**: `issue` + `refund` 건 합계
  - **누적 사용/소멸 포인트**: `use` + `revoke` + `expire` 건 합계
- **포인트 이용 내역 목록**:
  - 변동 일시, 구분(적립, 사용, 환불, 회수, 만료), 상세 내용(설명), 변동 금액(포인트), 만료 예정일(지급 건인 경우) 표시
  - **구분별 뱃지 스타일**:
    - `적립` (issue) / `환불` (refund): 파란색 또는 초록색 계열 뱃지, 플러스 기호 표시 (`+1,000 P`)
    - `사용` (use): 회색 계열 뱃지, 마이너스 기호 표시 (`-500 P`)
    - `회수` (revoke) / `만료` (expire): 빨간색 계열 뱃지, 마이너스 기호 표시 (`-2,000 P`)
  - **페이지네이션 및 필터**:
    - 포인트 내역 리스트 하단에 페이징 컴포넌트 추가
    - 내역 유형 필터(전체, 적립/환불, 사용, 소멸/회수) 지원

### C. 데이터 연동
- 기존 서비스 레이어 `pointService.getPointSummary(userId)` 및 `pointService.getPointTransactions(userId)`를 호출하여 실시간 로그인 사용자의 포인트 데이터를 바인딩합니다.

## 3. 검증 계획 (Verification Plan)
- **컴파일 빌드 검증**: `npm run build`를 구동하여 컴포넌트 및 라우팅 추가에 따른 타입 안전성과 빌드 성공 여부를 확인합니다.
- **인터랙션 및 UI 검증**: 마이페이지 네비게이션에서 '마이 포인트' 클릭 시 정확한 경로 이동 및 데이터 정합성 노출을 검증합니다.
