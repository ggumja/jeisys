# [PDCA Plan] 누락된 주문/배송/클레임 프로세스 개발 계획

본 계획은 제이시스 메디칼 B2B 시스템의 완성도를 높이기 위해, 표준 이커머스 프로세스 대비 현재 시스템에서 누락된 핵심 기능을 식별하고 이를 단계별로 개발하는 것을 목표로 합니다.

## 1. 현재 시스템 Gap 분석

### 1단계: 주문 및 입금
- [ ] **가상계좌 입금 독려 SMS**: 미입금 고객 대상 자동/수동 SMS 발송 기능 부재.
- [ ] **입금 기한 관리**: 설정된 기간(예: 3일) 경과 시 자동 주문 취소 로직 부재.
- [ ] **결제 데이터 검증**: 가입금 및 입금자명 불일치 건에 대한 관리자 확인 프로세스 보완 필요.

### 2단계: 배송 처리
- [ ] **주문서(Pick List) 출력**: 창고 전달용 통합 주문서 출력 및 상태 전환(상품준비중) 명확화.
- [ ] **외부 알림 연동**: 송장 채번 시 고객에게 알림톡/SMS 자동 발송 로직 부재.
- [ ] **실시간 배송 추적**: 운송장 번호 클릭 시 택배사 API 연동 실시간 위치 표시.
- [ ] **구매 확정 자동화**: 배송 완료 후 일정 기간(7~14일) 경과 시 자동 구매 확정 처리.

### 3단계: 취소/반품/교환 (클레임)
- [ ] **클레임 상태 확장**: `취소요청`, `반품요청`, `반품입고(검수중)`, `교환요청` 상태 추가.
- [ ] **클레임 관리 UI**: 관리자 페이지 내 클레임 전용 탭 및 상세 페이지 내 승인/거절 처리 화면 기능.
- [ ] **재고 자동 환원**: 반품 완료(입고 확인) 시 해당 상품 재고 자동 플러스(+) 처리.
- [ ] **교환/맞교환 로직**: 기존 상품 수거 확인 후 새 송장 자동 발행 및 추적 관리.

---

## 2. 단계별 개발 로드맵

### Phase 1: 클레임 인프라 및 상태 확장 (Short-term)
- **DB 스키마 수정**: `orders.status` 제약 조건 업데이트 및 클레임 관련 컬럼 추가.
- **클레임 관리 UI**: `OrderManagementPage` 내 클레임 필터 및 상세 페이지 내 승인/거절 버튼 구현.
- **환불 로직 통합**: 기존 `OrderCancelModal`을 확장하여 부분 반송 및 환불 금액 자동 계산 로직 적용.

### Phase 2: 알림 및 배송 워크플로우 최적화 (Mid-term)
- **알림 통합 (Notification Center)**: Aligo/Solapi 등 외부 SMS API 연동 인터페이스 구축.
- **물류 연동 보완**: 피킹 리스트(Pick List) 대량 출력 기능 및 '상품준비중' 상태 도입.
- **배송 추적**: 로젠택배 등 주요 택배사 API를 활용한 실시간 위치 정보 바인딩.

### Phase 3: 자동화 엔진 도입 (Long-term)
- **스케줄러 구현**: Supabase Edge Functions + pg_cron을 활용한 자동 작업(미입금 취소, 배송완료 전환, 구매확정) 배치 구축.
- **재고 모니터링**: 클레임 발생 시 실시간 재고 연동(Restocking) 자동화.

---

## 3. 사용자 확인 사항 (User Review Required)

> [!IMPORTANT]
> **1. 알림 서비스 연동**: 실제 SMS/알림톡 발송을 위해 선호하시는 서비스 업체가 있으신가요? (예: 알리고, 솔라피) 우선 Mock(가상) 연동으로 진행할까요?
>
> **2. 자동 취소/확정 정책**: 
> - 입금 대기 취소: 3일(72시간) 기준이 적당할까요?
> - 자동 구매 확정: 배송 완료 후 7일 기준이 적당할까요?
> 
> **3. 클레임 배송비 정책**: 반품 시 고객 과책일 경우 배송비(3,000원)를 환불액에서 자동 차감하는 로직을 포함할까요?

## 4. 검증 계획

### 자동화 테스트
- 미입금 주문 생성 후 72시간 경과 시 상태 변경 확인.
- 반품 입고 승인 시 상품 재고 수량 복구 확인.

### 수동 테스트
- 관리자 페이지에서 클레임 요청 건 승인/절차 테스트.
- 송장 출력 및 피킹 리스트 레이아웃 정합성 확인.
