# PDCA Plan: 회원관리 하위 크레딧 및 포인트 이력 관리 메뉴 추가와 조회 기능 구현

## 1. 개요 (Plan)
회원관리 메뉴 아래에 '크레딧 이력 관리' 및 '포인트 이력 관리' 하위메뉴를 신설하고, 쇼핑몰에 가입한 모든 회원의 크레딧/포인트 변동 내역(지급/충전, 사용/차감, 취소환불, 만료, 회수) 전체를 리스트로 상세히 조회할 수 있는 통합 관리자 화면을 각각 구현합니다.

## 2. 세부 사양 (Design)

### A. 메뉴 및 라우팅 구조
- **메뉴 구성**:
  - 기존 '회원관리' 단일 메뉴 -> 아코디언 메뉴로 변경
  - 하위 메뉴:
    1. 회원 목록 (`/admin/members`)
    2. 크레딧 이력 관리 (`/admin/credit-history`)
    3. 포인트 이력 관리 (`/admin/point-history`)

### B. 크레딧 및 포인트 이력 목록 테이블 명세
- **공통 컬럼**:
  - 일시 (날짜 및 시간)
  - 회원 아이디
  - 회원명 (병원명)
  - 구분 (지급/충전, 사용/차감, 환불, 회수, 만료) - 타입별 뱃지 적용
  - 변동액 (구분별 부호/색상 차별화, 크레딧은 원화 표시, 포인트는 P 단위 표시)
  - 상세 내용 / 메모
  - 관련 주문 번호 (주문 ID 링크 제공)

### C. 필터링 & 검색 (두 페이지 공통)
- **유형 필터**: 전체, 지급/충전, 사용/차감, 환불, 회수, 만료
- **검색어 필터**: 회원명, 병원명, 이메일, 아이디 검색

### D. 추가 기능 (두 페이지 공통)
- **Excel 다운로드**: XLSX 라이브러리 연동을 통한 전체 조회 내역 엑셀 다운로드
- **페이지네이션**: 10, 20, 50, 100개 단위 페이징 처리 및 인덱스 정렬
