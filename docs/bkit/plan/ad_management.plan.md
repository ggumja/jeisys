# [Plan] 광고 관리 시스템 (Advertisement Management System)

## 1. 개요 (Overview)
홈페이지 내 다양한 위치(메인 배너, 이메일 배너, 팝업, 사이드 배너 등)에 게시되는 광고 콘텐츠를 효율적으로 관리하고, 사용자의 반응(노출, 클릭)을 추적하여 리포트로 제공하는 통합 광고 관리 시스템을 개발합니다.

## 2. 목표 (Goals)
- **통합 관리**: 다양한 광고 지면을 하나의 Admin UI에서 관리.
*   **유연한 스케줄링**: 예약 게시 및 종료 기능.
*   **성과 추적**: 광고별 노출수(Impression), 클릭수(Click), 클릭률(CTR) 자동 집계.
*   **데이터 시각화**: 기간별/광고별 성과 리포트 제공.

## 3. 주요 기능 (Key Features)

### 3.1. Admin 광고 관리 (Management)
- **광고 등록/수정**: 제목, 이미지(PC/Mobile), 링크 URL, 노출 순서, 기간 설정(시작일/종료일).
- **지면 분류 (Placement)**:
    - 메인 배너 (Main Banner)
    - 이메일 배너 (Email Banner)
    - 팝업 광고 (Popup)
    - 페이지 사이드 배너 (Side Banner)
- **상태 제어**: 활성/비활성 토글.

### 3.2. 추적 시스템 (Tracking)
- **노출 추적**: 광고가 화면에 렌더링될 때 카운트 (Intersection Observer 사용 권장).
- **클릭 추적**: 광고 클릭 시 목적지 URL 이동 전 카운트.
*   **추적 파라미터**: UTM 파라미터 자동 생성 기능 (링크 이동 시 성과 분석용).

### 3.3. 리포트 (Reporting)
- **대시보드**: 전체 광고 성과 요약 (오늘의 노출/클릭).
- **상세 리포트**: 기간 필터링(일/주/월), 광고 지면별 비교 차트.
- **데이터 추출**: 성과 데이터 CSV 내보내기.

## 4. 데이터 구조 계획 (Data Schema)
- `ads`: 광고 기본 정보 및 설정 테이블.
- `ad_stats`: 일별 노출/클릭 집계 테이블 (성능을 위해 로그 기반이 아닌 집계 기반 권장).

## 5. 추진 일정 (Schedule - PDCA)
1. **Plan**: 현재 단계 (기획 및 요구사항 정의)
2. **Design**: DB 스키마 설계 및 UI 와이어프레임 설계
3. **Do**: Admin 관리 기능 구현 -> 추적 서버 로직 구현 -> 리포트 시각화 구현
4. **Check**: 추적 데이터 정확도 검증 및 UI 편의성 테스트
5. **Act**: 실사용 피드백 반영 및 고도화

## 6. 기대 효과 (Expected Outcomes)
- 데이터 기반의 마케팅 의사결정 가능.
- 수동 배너 교체 작업의 번거로움 해소 및 자동화.
