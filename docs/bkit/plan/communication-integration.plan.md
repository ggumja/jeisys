# PDCA Plan: 커뮤니케이션 통합 시스템 구현 (FAQ, 메뉴얼, 뉴스, 미디어, 데모신청)

## 1. 개요
- **목표**: 1:1 문의 외의 나머지 커뮤니케이션 메뉴들을 실제 DB와 연동하여 관리 효율성을 높인다.
- **범위**: FAQ, 매뉴얼, 뉴스, 미디어, 장비 데모신청

## 2. 요구사항
- 각 기능별 목록/상세 페이지 실시간 연동.
- 관리자 페이지에서의 CRUD 기능 제공.
- 뉴스/미디어의 경우 조회수 추적 기능.
- 데모신청의 경우 신청-확정-완료 프로세스 관리.

## 3. 기술 설계
- **Services**:
    - `postService.ts`: FAQ, Manual, News, Media 통합 관리 (PostType 구분)
    - `demoService.ts`: 장비 데모신청 전용 관리
- **Database**:
    - `posts` 테이블: `type` 컬럼을 통해 FAQ, 뉴스, 미디어, 매뉴얼 구분.
    - `demo_requests` 테이블: 신규 생성 (병원명, 연락처, 장비명, 일정 등).
